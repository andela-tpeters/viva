    Introduction

    Welcome to the Viva API.

    Viva is an OAuth2 provider which exposes various endpoints to allow application for create, retrieve, update and delete bucketlists on authorization by the user.

    On the right is an example terminal containing various code examples that shows you how to better use the API.

    On the left are navigation helpers to help you search this document or jump through to specific sections.

    The base url for the API is Viva

    Current version of API is version 1

    Using OAuth2

    Initialize new client

    Create new client

    client = OAuth2::Client.new(
                            YOUR_CLIENT_ID,
                            YOUR_CLIENT_SECRET,
                            site: "http://viva.herokuapp.com",
                            authorize_url: "/oauth/authorize",
                            token_url: "/oauth/token"
                            )

    First you need oauth2 gem:

    Add this line to your Gemfile:

    gem 'oauth2'

    On the terminal, cd into the project directory and run:

    bundle install

    Authorization url

    Authorization url

    authorization_url = client.auth_code.authorize_url(
                                    redirect_uri: YOUR_CALLBACK_URI
                                  )

    The link returned by the authorize_url should be used in views. When users click on the link, they will be prompted to grant access to the application that is trying to access their bucketlists

    When authorization is successful, viva redirects back to the redirect uri provided with a code query.

    Authorization url Http Response

    http://YOUR_CALLBACK_URI?code=VIVA_GENERATED_CODE

    Access Token

    This token can be stored in sessions. The token is used to make authorized calls to viva api endpoints

    To get Access Token

    token_object = client.auth_code.get_token(
                                    VIVA_GENERATED_CODE,
                                    redirect_uri: YOUR_CALLBACK_URI
                                  )

    Token

    You can view the token with this command token_object.token

    The VIVA_GENERATED_CODE returned after user has granted access to the application to access their bucketlists is used to get access token that is used to interact with the api endpoints

    Your callback uri is the route you want Viva to redirect to after user authorization

    Using RestClient

    Install RestClient

    # On the terminal, cd into the project directory and run:
bundle install

    Add to your Gemfile

    gem 'rest-client'

    Authorization Url

    Authorization url for users to authorize the app:

    http://viva.herokuapp.com/oauth/authorize?client_id=YOUR_APP_ID&redirect_uri=YOUR_REDIRECT_URI&request_type=code

    Authorization url Http Response

    http://YOUR_CALLBACK_URI?code=VIVA_GENERATED_CODE

    Access token

      req_params = "client_id=YOUR_APP_ID&\
                client_secret=YOUR_APP_SECRET&\
                code=VIVA_GENERATED_CODE&\
                grant_type=authorization_code&\
                redirect_uri=YOUR_REDIRECT_URI\
              "
  response = JSON.parse RestClient.post(
              "http://viva.herokuapp.com",
              req_params
            )

  token = response["access_token"]
    The access token can be stored in sessions and would be required for making requests to the endpoints

    Requests

    Using OAuth2 with Access Token Object

    # initialize access token
@client = OAuth2::Client.new(APP_ID, APP_SECRET, site: "http://viva.herokuapp.com" )

token_object = OAuth2::AccessToken.from_hash(
                  @client, { access_token: session[:access_token] }
                )

    GET Requests

    response = JSON.parse token_object.get(API_ENDPOINT)

    curl API_ENDPOINT?access_token=YOUR_ACCESS_TOKEN

# OR

curl API_ENDPOINT -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

    POST Requests

     response = JSON.parse token_object.post(
                API_ENDPOINT, params: { name: "bucketlist_name" }
              )

    curl -X POST API_ENDPOINT -d "name=bucketlist_name"\
     -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

    OAuth requests involves initializing an access token object with the access token gotten from authorization and then using the access token object to make requests

    Initialize Access Token

    @client = OAuth2::Client.new(APP_ID, APP_SECRET, site: "http://viva.herokuapp.com" )

    token_object = OAuth2::AccessToken.from_hash( @client, { access_token: session[:access_token] } )

    Using RestClient with Access Token query

    GET Requests

    response = JSON.parse RestClient.get(API_ENDPOINT)

    curl API_ENDPOINT?access_token=YOUR_ACCESS_TOKEN

# OR

curl API_ENDPOINT -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

    POST Requests

    params = "name=bucketlist_name&access_token=YOUR_ACCESS_TOKEN"

response = JSON.parse RestClient.post(API_ENDPOINT, params)

    curl -X POST API_ENDPOINT -d "name=bucketlist_name"\
     -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

    With Rest Client, requests are made with passing the access token as a query params

    Endpoints

    All endpoint call should be to http://viva.herokuapp.com/api/v1/
    Endpoint Functionality Public Access
    POST /bucketlists Create Bucketlist FALSE
    GET /bucketlists All Bucketlists FALSE
    GET /bucketlists/:id Show Bucketlist FALSE
    PUT /bucketlists/:id Update Bucketlist FALSE
    DELETE /bucketlists/:id Delete Bucketlist FALSE
    POST /bucketlists/:id/items Create bucketlist item FALSE
    GET /bucketlists/:id/items All bucketlist items FALSE
    GET /bucketlists/:id/items/:item_id Show bucketlist item FALSE
    PUT /bucketlists/:id/items/:item_id Update bucketlist item FALSE
    DELETE /bucketlists/:id/items/:item_id Delete bucketlist item FALSE

    Bucketlists

    Create New Bucketlist

    Request format

    curl X POST http://viva.herokuapp.com/api/v1/bucketlists \
    -d "name=bucketlist_name" \
    -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

    response = token.post("http://viva.herokuapp.com/api/v1/bucketlists",
            { name: "bucketlist_name" }
            )

  "items": [],
  "date_created": "2015-08-12 11:57:23",
